Mood-Based Music Discovery Website

Project Overview

This project is a playful website that recommends songs matching the user’s mood of the day. Inspired by my love for music and the challenge of finding the right vibe, I wanted to create a light, interactive experience that encourages users to reflect on how they feel and discover new music in a fun way. Rather than building a traditional, practical tool, the focus was on creativity and engagement.

  • Focus: Personalized mood-based song recommendations
  • Audience: Casual music listeners exploring new tracks
  • Goal: Deliver a fun and intuitive music discovery experience

What I’d do differently: This project was developed and deployed using Glitch as part of the coursework requirements. While Glitch is primarily used in educational or prototyping contexts, it helped rapid development and live testing. In future projects, I would deploy using other platforms for greater scalability and performance.

Design & Functionality Overview

  • Users choose a mood or take a quiz to find their emotional state
  • Receive a song recommendation with options to listen on Spotify, Apple Music, or YouTube
  • Refresh recommendation or restart quiz for new suggestions
  • Share personalized song links with friends
  • Discovery section for browsing artists with info, popular releases, and videos
  • Minimal, clean design with a calming animated gradient on homepage
  • Color palette combines blue (calm) and orange (energy) to reflect moods
  • Typography balances readability and personality using Work Sans and Bebas Neue
  • Dynamic JavaScript loading for personalized, engaging interactions (loading messages, mood popups)
  • Simple quiz algorithm matches answers to moods using data attributes and tallying
  • Song data stored externally in JSON for better scalability
  • Randomized, non-repeating song selection with logic to avoid repeats on refresh
  • Background colors change according to mood on the song display page

What I’d do differently: Develop a fully dynamic artist discovery section and unify CSS files for easier management.

Reflection & Future Improvements

  • Created an engaging, playful platform that invites mood-based music exploration
  • Limited song dataset limits variety; API integration (e.g., Spotify) would improve content depth
  • Future plans include user accounts to save favorites and social sharing features
  • Mobile compatibility needs improvement, especially for features like copy-to-clipboard
  • Design can be enhanced with richer visuals while maintaining minimalism

What I’d do differently: Conduct mobile-first testing earlier and gather user feedback to optimize quiz accuracy and UX.
