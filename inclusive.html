Accessibility Audit – RBO Website

Project Overview

As part of a university brief, I conducted an expert accessibility audit for the Royal Ballet and Opera’s website, focusing on two key pages: the homepage and visitor information section. The goal was to identify severe accessibility issues and provide actionable fixes to help RBO better serve users with access needs.

Approach & Tools

  • Guidelines referenced: WCAG 2.0, 2.1, and 2.2 (Levels A & AA)
  • Testing tools: WAVE, Lighthouse, WebAIM Contrast Checker, ColorZilla
  • Assistive tech: macOS VoiceOver (screen reader)
  • Methodology: Manual testing and persona-based review

Key Personas Considered

  • Screen reader users (blind/low vision)
  • Keyboard-only users (motor impairments)
  • Users with low vision or color blindness
  • Visitors with cognitive load sensitivity

Top 6 High-Severity Issues

  • No “Skip to Content” link: Prevents quick navigation for keyboard/screen reader users
    Fix: Add a skip link at the top of the page.
  • No audio description for homepage video: Inaccessible for blind or low vision users
    Fix: Add an audio description or transcript.
  • Poor focus visibility: Hard to track for keyboard users
    Fix: Increase contrast of focus outlines (minimum 3:1 ratio).
  • Unlabeled search icon: Not announced by screen readers
    Fix: Add aria-label="Search".

Medium and Low Severity Issues

  • Long image alt text: Overwhelming for screen readers
    Fix: Shorten alt text to under 100 characters where possible.
  • Heading structure skips levels: Disrupts document hierarchy for screen readers
    Fix: Use proper heading levels sequentially (e.g., no skipping from H2 to H4).

Recommendations

  • Address high-severity issues first, especially those impacting navigation and media
  • Conduct user testing with assistive tech users to validate changes
  • Embed accessibility into future design systems
  • Implement regular compliance audits as content evolves

Reflection

This project reinforced the importance of accessibility in public-facing cultural institutions. In a business setting, I would supplement this audit with a collaborative workshop involving designers and developers, and ensure remediation is tracked across sprints with proper testing coverage. Accessibility isn’t a one-off task — it’s a continuous practice.
