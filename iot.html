NeuroSync: Wearable Seizure Alert Bracelet & App

Project Overview

NeuroSync is an IoT-enabled wearable bracelet paired with a mobile app designed to detect seizures and alert emergency contacts, while providing ongoing health monitoring and personalized support for epilepsy management.

Technical Implementation

The bracelet integrates multiple sensors — EDA, accelerometer, gyroscope, peripheral temperature, and heart rate — connected to an Arduino board. Data streams in real-time to a smartphone app, which analyzes signals and sends alerts if seizure activity is detected.

  • Electrodermal Activity (EDA) sensor for emotional and physiological signals
  • Motion sensors (accelerometer & gyroscope) to detect abnormal movements
  • Peripheral temperature & heart rate sensors for vital sign monitoring
  • Arduino-based prototype for sensor integration and data collection
  • Mobile app backend processes sensor data and manages alerts

Key Use Cases & User Interaction

  • Automatic seizure detection through combined sensor data
  • User notifications with option to cancel false alarms
  • Emergency contact alerts when seizures are confirmed
  • Health tracking and personalized management advice

Business & User Research

The project included detailed business model development, user research, and market analysis to ensure the product addresses real-world needs and delivers value to both users and stakeholders.

  • Identification of key customer segments
  • Validation of market needs and competitive landscape
  • Value proposition and revenue streams analysis

Reflection

This interdisciplinary project enhanced my skills in IoT integration, embedded systems, user-centered design, and business strategy — all essential for developing impactful tech solutions in healthcare.

Project Context & Learnings

This project was completed as part of a university course, aiming to deliver a holistic approach combining technical development, business analysis, and user-centered design.

If I were to develop NeuroSync in a professional business setting, I would prioritize deeper market validation, secure partnerships with healthcare providers, and implement a more scalable backend infrastructure to support real-time data processing and compliance with medical regulations.

Although our analysis was comprehensive for an academic project, real-world deployment would also require extensive clinical testing and regulatory approvals.

Source Code & More

Due to time constraints, the Arduino implementation concentrated primarily on the heart rate sensor. While we aimed for full integration of all sensors, the available code currently covers only heart rate monitoring and is not available for sharing at this time.

*Note: The Arduino-based sensor data collection and the mobile app were developed as separate components and have not yet been fully integrated. Future work would focus on creating seamless communication between the hardware and the application to enhance real-time data processing and user experience.
